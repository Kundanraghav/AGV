so in cron hourly there is a file which has write access to www-data
this file is being run by root but has write access to us, making it a entry point to
get root privileges

so I edited the file /etc/cron.hourly/cuiteur-cleaning to grant myself root access
as follows

echo '#!/bin/bash' > /etc/cron.hourly/cuiteur-cleaning
rewrites the whole file which the mentioned under ''(rewrites cuz we're using >
appends if we're using >>)

sed -i 's#www-data:x:33:33:www-data:/var/www:/usr/sbin/nologin#www-data:x:0:0:root:/root:/bin/bash#' /etc/passwd
echo "www-data:x:33:33:www-data:/var/www:/usr/sbin/nologin" >> /etc/passwd
echo "www-data:password123" | chpasswd

echo these three commands into the cuiteur cleaning file and chmod -x to make it 
executable again

the first command uses sed to find and replace the current uID of www-data with the
same uID as root in the file etc/passwd, where all user and user priveleges are stored
the second command appends the command to assign www-data the old uID the device 
remembers, so this makes www-data has 2 uIDs, making that user act as root and normal
third command adds a password to the root login for www-data and the password I set
was password123. Wait for this cron job to run and then once it run try with the 
following command
